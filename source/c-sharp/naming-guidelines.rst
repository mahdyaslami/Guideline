خط مشی نام گذاری
================

:doc:`English <naming-guidelines-en>`

پیروی کردن از یک مجموعه قرارداد نامگذاری در توسعه یک framework می تواند سهم عمده
ای در قابلیت استفاده مجدد از آن framework داشته باشد. این امر به framework این 
اجازه را می دهد که توسط بسیار از توسعه دهندگان و در پروژه های گوناگون قابل 
استفاده باشد. ورای داشتن یک فرم ثابت، نام های اجزای یک framework باید به سادگی 
قابل فهم باشد و عملیاتی که آن جزء انجام می دهد را به کاربر انتقال دهد.

هدف این فصل مشخص کردن یک مجموعه قواعد و قرارداد های نامگذاری است که نتیجه آن 
ایجاد یک حس خوب در توسعه دهنده هاست.

اگرچه استفاده از این قرارداد ها در کل پروژه شما یک ثبات و استحکام در نامگذاری در
کد های شما بوجود می آورد ولی ضرورت استفاده از این قواعد بیشتر در API هایی است که
به صورت عمومی انتشار داده شده اند (انواع و اعضای عمومی (public) و محافظت شده 
(protected) و interface هایی که صریحا پیاده سازی شده اند).
